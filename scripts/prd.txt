# Overview  
SSOM(System Status & Operations Monitoring)은 금융 도메인에 특화된 실시간 시스템 관제 앱입니다. 운영 안정성을 확보하고 장애 대응 속도를 향상시키기 위해 개발되었습니다. 실시간 메트릭 시각화, AI 기반 장애 분석, 부서별 권한 제어 등을 통해 운영팀의 효율적인 모니터링 환경을 제공합니다.

# Core Features  
- **사용자 인증 및 권한 관리**: 사내 인증 시스템 연동 로그인, 첫 로그인 시 비밀번호 변경, 부서별 접근 권한 설정(로그/메트릭 접근 범위 제어)
- **실시간 대시보드**: HTTP 요청 수, 응답 시간, JVM 메모리, GC 빈도 등 핵심 지표 시각화, 컨테이너 수준 CPU/메모리 사용량, 실시간 데이터 자동 갱신(SSE), 무사고 일수 시각화
- **알림 시스템**: Prometheus Alertmanager 기반 경고, FCM 푸시 알림, SSE 기반 실시간 이벤트 스트리밍(읽음 처리 포함)

# User Experience  
- **User Personas**: 운영팀(모니터링/장애 대응), 개발팀(로그/지표 분석), 관리자(권한 관리)
- **Key User Flows**:
  1. 사내 계정 로그인
  2. 첫 로그인 시 비밀번호 변경
  3. 대시보드에서 본인 부서 시스템의 지표/로그 확인
  4. 장애 발생 시 알림 수신

- **UI/UX Considerations**: 실시간성, 부서별 데이터 필터링, 모바일 친화적 UI(React Native 기반), 접근성

# Technical Architecture  
- **Frontend**: Expo(React Native) 기반 모바일 앱
  - 모바일 우선(Mobile-first) 설계, Android 지원
  - 실시간 데이터 반영(SSE 기반)
  - 주요 화면: 로그인, 비밀번호 변경, 대시보드(지표/로그), 알림 센터, AI 장애 분석 결과
  - 부서별 데이터 필터링 및 접근 제어 UI
  - 접근성 및 반응형 UI/UX
  - FCM을 통한 푸시 알림 처리
  - 대시보드 내 실시간 차트 및 무사고 일수 시각화

# Development Roadmap  
- **MVP Requirements**:
  - 사내 인증 연동 및 권한 관리
  - 실시간 대시보드(핵심 지표, SSE)
  - 알림 시스템(경고 수신, SSE/FCM)
- **Future Enhancements**:
  - 공지사항 배너
  - 메트릭/로그 상세 항목 확정 및 확장
  - 장애 분석 결과 기반 보고서 자동 생성
  - 외부 서비스 통합(Slack, Jira 등)

# Logical Dependency Chain
- 1단계: 인증/권한 관리 → 대시보드(지표/로그) → 알림 시스템
- 2단계: AI 장애 분석(기본) → 무사고 일수 시각화
- 3단계: 공지사항, 외부 서비스 연동, 보고서 자동화 등
- **우선순위**: 운영팀이 즉시 활용 가능한 실시간 대시보드와 알림 시스템을 최우선 구축

# Risks and Mitigations  
- **기술적 난이도**: 실시간 데이터 처리(SSE), AI 장애 분석 정확도
- **MVP 범위 결정**: 핵심 기능에 집중, 부가 기능은 후순위
- **리소스**: 사내 인증/보안 정책, 외부 서비스 연동 시 정책 이슈
- **대응 방안**: 단계별 개발, PoC 통한 검증, 사내 정책 준수

# Appendix  
- **메트릭/로그 상세 항목**: HTTP 요청/응답, JVM, GC, 컨테이너 리소스, 서비스 오류, 인증/인가, 사용자 행동, 알림 트래킹 등
- **향후 고려 사항**: 공지사항, 외부 서비스 연동, 보고서 자동화 등

# Frontend Requirements
- **기술 스택**: Expo(React Native) 기반 모바일 앱 (iOS/Android 동시 지원)
- **권장 라이브러리**: @mui/material @emotion/react @emotion/styled, React Native SSE, React Native Gifted Charts
- **모바일 우선(Mobile-first) 설계** 및 접근성/반응형 UI/UX
- **주요 화면 및 기능**:
  - 로그인/인증: 사내 인증 시스템 연동, 첫 로그인 시 비밀번호 변경, 부서별 권한에 따른 UI/데이터 제어
  - 실시간 대시보드: HTTP 요청 수, 응답 시간, JVM 메모리, GC 빈도, 컨테이너 리소스 등 주요 지표 실시간 시각화(SSE 기반), 무사고 일수 표시
  - 알림 센터: Prometheus Alertmanager 기반 경고, FCM 푸시 알림, SSE 실시간 이벤트 스트리밍(읽음 처리 포함), 알림 트래킹
- **UI/UX 고려사항**:
  - 부서별 데이터 필터링 및 접근 제어 UI
  - 실시간성 강조(데이터 지연 최소화)
  - 모바일 친화적, 직관적 네비게이션
  - 장애 발생 시 즉각적인 알림 및 분석 결과 노출
  - 향후 공지사항 배너, 외부 서비스 연동(Slack, Jira 등) 확장 고려
